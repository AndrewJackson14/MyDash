# Messaging, notifications, and MyHelper

**Messages** (**Ops → Messaging**) is internal chat between team members. Every message lives in `team_notes` and is tied to a sender, recipient, optional context (an entity like a client or issue), and read state.

**Notifications** (top-right bell icon) surface:
- Unread messages
- New inquiries (if you're in sales)
- Approaching deadlines
- Sale confirmations, proposal signatures, invoice events (per your alert preferences)

Click **Alert preferences** in your profile to toggle which events notify you.

## MyHelper — the internal assistant

MyHelper is a chatbot that answers "how do I..." and "where is..." questions about MyDash. It reads from the MyDash docs corpus and responds with grounded answers.

**Two ways to reach MyHelper:**

- **Floating button** — the 🤖 bubble in the bottom-right of every page. Press `⌘/` (Mac) or `Ctrl+/` (Windows) to open.
- **Messages** — MyHelper is pinned to the top of the team picker. Message it like you'd message any teammate.

**What it can answer:** workflow questions (how to build a proposal, how to send an invoice), navigation questions (where does X live), basic how-tos from the docs.

**What it can't:** live data ("what did client X spend?"), financial advice, personnel decisions, anything not in the docs. For those it pings **MySupport** — whoever currently holds the MySupport role (right now: Andrew).

**If MyHelper gives you wrong info**, message MySupport and the docs will get updated. Feedback closes the loop.
